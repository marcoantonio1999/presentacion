## 1. Visión General



Claro Video Web es una **SPA (Single Page Application)** construida sobre React 17, con Redux para gestión de estado, redux-saga para efectos asíncronos, y SSR parcial mediante Express + Handlebars.

## 2. Capas de la Aplicación



### Capa 1: Presentación

```

src/react/components/      ← Componentes puros (sin Redux, sin lógica de negocio)

src/react/newComponents/   ← Nuevos componentes (migración activa)

src/react/layouts/         ← Wrappers de página (header, footer, grids)

```

- Sin acceso directo al store

- Comunicación solo por `props`

- Sin llamadas HTTP



### Capa 2: Contenedores

```

src/react/containers/   ← 50+ módulos conectados a Redux

src/react/hooks/         ← Lógica reutilizable encapsulada

src/react/context/       ← React Context (estado no-Redux)

src/react/providers/     ← Providers de alto nivel (UserContext)

```

- Usan `useSelector` / `useDispatch` (nuevo código)

- Usan `connect()` solo en código legado

- Orquestan la lógica de UI



### Capa 3: Estado (Redux)

```

src/react/store.js       ← Configuración del store

src/react/actions/       ← Action creators síncronos

src/react/selectors/     ← Selectores memoizados (Reselect)
```

- Estado centralizado y predecible

## 3. Flujo de Datos Redux



```

Usuario → dispatch(action) → Middleware → Saga → API → Reducer → Selector → UI

```



1. El usuario interactúa con un componente

2. El componente llama `dispatch(myAction.request(payload))`

3. El middleware ejecuta thunks/sagas

4. La saga llama a la API con `call(apiService, params)`

5. La respuesta dispara `put({ type: SUCCESS, payload })`

6. El reducer actualiza el estado

7. El selector memoizado devuelve los datos procesados

8. El componente re-renderiza

## 4. Patrón de Módulo Feature



Cada feature grande sigue esta estructura:



```

containers/MyFeature/

├── index.js           ← Componente principal (conectado)

├── actions.js         ← Action creators del feature

├── reducer.js         ← Reducer del slice

├── saga.js            ← Effects async

├── selectors.js       ← Selectores Reselect

├── constants.js       ← Constantes locales

└── components/        ← Sub-componentes presentacionales

```

### Patrones Arquitectónicos Principales



- **Flux/Redux:** Estado unidireccional centralizado — `src/react/store.js`, `reducers/`, `actions/`

- **Container/Presenter:** Separación estado/UI — `containers/` conectados vs `components/` puros

- **Saga Pattern:** Efectos asíncronos como generadores — `appSaga.js`, saga por feature en `containers/`

- **Selector Memoization:** Reselect obligatorio para derivaciones — `src/react/selectors/`

- **Module Pattern:** Features encapsuladas — `src/espn/`, `src/hbo/`, `src/ExternalLoginApps/`

- **Code Splitting:** Chunks por ruta — Webpack `splitChunks: all` en `config/webpack.prod.js`



### Principios de Diseño



- **Unidireccionalidad:** El flujo de datos siempre va en una sola dirección: Action → Reducer → Store → Selector → UI

- **Separación de responsabilidades:** Cada capa tiene un rol definido; los componentes no acceden a la API directamente

- **Inmutabilidad:** Los reducers nunca mutan el estado; siempre retornan nuevos objetos

- **Evidencia:** `src/react/store.js`, `src/react/appSaga.js`, `config/webpack.prod.js`

## 🔄 Patrones de Diseño



### Patrón Arquitectónico



**Flux/Redux** — arquitectura unidireccional de estado. El estado global vive en un único store (`src/react/store.js`). Las acciones describen cambios, los reducers los aplican, los selectors los consumen.



```javascript

// Evidencia: src/react/store.js

const store = createStore(

  persistReducer(persistConfig, rootReducer),

  composeEnhancers(applyMiddleware(...middlewares))

);

```



### Dependency Injection



No se usa framework de DI explícito. Inyección de dependencias vía:

- **Redux hooks:** `useSelector(selector)` / `useDispatch()` para estado global

- **React Context:** `useContext(UserContext)` para datos de usuario — `providers/user.provider.js`

- **Props drilling:** En componentes presentacionales pequeños de `components/`



### Repository Pattern



La capa `src/react/api/services/` actúa como repositorios por dominio:

```

api/services/

├── content/   ← Repositorio de contenido VOD/Live

├── user/      ← Repositorio de usuario y autenticación

├── player/    ← Repositorio de streaming (URLs, DRM)

└── ...        ← Un repositorio por dominio de negocio

```



### Observer/State Management



- **Redux:** Store centralizado con suscripciones vía `useSelector`

- **Reselect:** Selectores memoizados en `src/react/selectors/` — recalculan solo cuando cambian inputs

- **redux-persist:** Persiste slices del estado en cookies vía `createCookieMiddleware` — `src/react/store.js`

## 🛡️ Consideraciones de Seguridad



### Autenticación y Autorización



- **Mecanismo principal:** Tokens JWT almacenados en cookies vía `createCookieMiddleware` (`src/react/store.js`)

- **OAuth social:** Facebook (`react-facebook-login`), Apple (`react-apple-login`), ESPN (`src/espn/`), HBO (`src/hbo/`)

- **Flujo de auth:** `Login Container → dispatch(loginRequest) → Saga → UserAPI → token → Cookie persist`

- ⚠️ No almacenar tokens en `localStorage` sin encriptar



### Manejo de Secrets



- **Variables de entorno:** `.env.development` y `.env.production` — inyectadas via `webpack.DefinePlugin`

- 🔴 **Riesgo crítico:** Archivos `.env.*` potencialmente versionados en el repositorio — agregar a `.gitignore`

- **Regla:** Nunca hardcodear URLs de API — usar constantes de `src/react/const/` o `src/react/constants.js`



### Validación de Datos



- **HTML del CMS:** Usar `react-html-parser` con precaución — validar y sanitizar con DOMPurify antes de renderizar

- **Inputs del usuario:** Validar en el lado del cliente antes de dispatch; validación definitiva en servidor

- **Protección XSS:** React escapa automáticamente en JSX; el riesgo aparece al usar `dangerouslySetInnerHTML`



### Protección de APIs



- **Interceptors Axios:** `src/react/api/interceptors/` — agregan headers de autenticación en cada request

- **Retry logic:** Configurado en interceptors para manejo de fallos de red

- 🔴 **Riesgo:** `axios@0.15.2` tiene múltiples CVEs activos (SSRF, DoS, CSRF, ReDoS) — actualizar a `^1.7.x`

## 3. Redux-Saga: Side Effects



```javascript

// ✅ PATRÓN DE SAGA ESTÁNDAR

import { call, put, takeLatest } from 'redux-saga/effects';

import { FETCH_CONTENT } from '../actions/contentActions';

import { getContent } from '../api/services/content';



function* handleFetchContent({ payload }) {

  try {

    const result = yield call(getContent, payload.id);

    yield put({ type: `${FETCH_CONTENT}_SUCCESS`, payload: result });

  } catch (error) {

    yield put({ type: `${FETCH_CONTENT}_FAILURE`, error });

  }

}



export default function* contentWatcher() {

  yield takeLatest(`${FETCH_CONTENT}_REQUEST`, handleFetchContent);

}

```



### Reglas de sagas



- Usar `takeLatest` (no `takeEvery`) para prevenir requests duplicados

### components/: Responsabilidades



- **Rol:** Renderizado visual puro

- **Restricciones:** Sin Redux, sin API calls, sin lógica de negocio

- **Comunicación:** Solo via `props` (recibe datos) y callbacks (emite eventos)

- **Ejemplo:** `components/footer/Footer.js`, `components/modal/ModalFactory.js`



### containers/: Responsabilidades



- **Rol:** Conectar UI con estado Redux y orchestrar acciones

- **Permite:** `useSelector`, `useDispatch`, lógica de ciclo de vida via `useEffect`

- **50+ módulos:** `Header/`, `Player/`, `Login/`, `EPG/`, `Music/`, `Payment/`, etc.

### api/services/: Responsabilidades



- **Rol:** Única capa que se comunica con APIs REST externas

- **Restricciones:** Solo HTTP — sin lógica de negocio ni transformaciones complejas

- **Nueve dominios:** `content`, `user`, `epg`, `payway`, `plan`, `player`, `metrics`, `nav`, `campaign`

### espn/ + hbo/ + ExternalLoginApps/: Responsabilidades



- **Rol:** Módulos standalone de autenticación externa

- **Aislamiento:** Tienen su propio estado Redux, services, constants, utils y components

- **Integración:** Se integran con el store principal solo via Redux actions definidas



### selectors/: Responsabilidades



- **Rol:** Derivar y memoizar datos del estado global

- **Regla:** Siempre usar `createSelector` (Reselect) para selectores derivados

- **Beneficio:** Evitan re-renders innecesarios al memoizar resultados

## Conceptos Clave para Entender la App



### 1. Routing basado en país



Toda la app usa rutas `/:country/{módulo}`. El `:country` es el código del país (ej: `ar`, `cl`, `mx`). La función `GetRegion()` de `constants.js` obtiene el país actual.



### 2. Estado global con Redux



El estado global se divide en slices (reducers). Cada container típicamente tiene su propio slice. Nunca accedas al estado directamente — usa selectores Reselect.



### 3. Efectos async con Sagas



Toda llamada a API va por una saga. Las sagas se conectan al store a través de `appSaga.js` (root saga). Cada container puede tener su propia saga registrada localmente.



### 4. UserContext



`providers/user.provider.js` provee `user_id` y `acceptTerms` a toda la app. Accedido via `useContext(UserContext)` o `static contextType = UserContext` (legado).



### 5. Módulos ESPN/HBO standalone



Son módulos relativamente independientes en `src/espn/` y `src/hbo/`. Tienen su propio estado, servicios y componentes.

## 🔧 Troubleshooting



### Errores Comunes



| Problema | Solución Rápida |

|----------|-----------------|

| `npm start` falla en Windows | Configurar `$env:npm_config_script_shell` con ruta a `bash.exe` de Git |

| Error certificado HTTPS | Instalar certificado en Trusted Root (ver TROUBLESHOOTING.md §2) |

| `local.clarovideo.com` no resuelve | Agregar entrada en `hosts` (ver `## ⚙️ Configuración del Entorno`) |

| Peer dependencies conflict | Usar `npm install --legacy-peer-deps` |

| Heap out of memory | `set NODE_OPTIONS=--max_old_space_size=4096` |

| Puerto 3000 en uso | Cambiar en `server/port.js` o `PORT=3001 npm start` |



### Performance Issues



- **Build lento:** Verificar que `node_modules/.cache/` existe (cache de Webpack) — si fue eliminado, el primer build tarda más

- **Tests lentos:** Usar `--maxWorkers=50%` para limitar workers en máquinas con poca RAM

- **HMR no funciona:** Verificar que el host `local.clarovideo.com` resuelve correctamente y el certificado está instalado



Ver [TROUBLESHOOTING.md](./TROUBLESHOOTING.md) para solución detallada de todos los errores.

## 1. `npm start` Falla en Windows (Exit Code 1)



**Síntoma:**

```

npm error Exit status 1

```



**Causa:** npm en Windows usa `cmd.exe` por defecto, pero los scripts del proyecto requieren bash.



**Solución:**



```powershell

# Opción A: Configurar para la sesión actual (PowerShell)

$bash = "C:\Program Files\Git\bin\bash.exe"

$env:npm_config_script_shell = $bash

npm start



# Opción B: Configurar permanentemente en el perfil PowerShell

# Agregar al archivo $PROFILE:

$env:npm_config_script_shell = "C:\Program Files\Git\bin\bash.exe"

```



**Prerrequisito:** Git for Windows instalado (incluye `bash.exe`).

## 2. Error de Certificado HTTPS en el Navegador



**Síntoma:** El navegador muestra "Tu conexión no es privada" al abrir `https://local.clarovideo.com:3000`.



**Solución:** 

- Es esperado — el certificado es auto-firmado en desarrollo

- Hacer clic en "Avanzado" → "Continuar a local.clarovideo.com (no seguro)"

- Para evitarlo repetidamente, agregar excepción permanente en el navegador

## 3. `local.clarovideo.com` No Resuelve (ERR_NAME_NOT_RESOLVED)



**Síntoma:** El navegador no puede conectar a `local.clarovideo.com`.



**Causa:** El host no está configurado en el archivo `hosts`.



**Solución Windows** (ejecutar como Administrador):

1. Abrir `C:\Windows\System32\drivers\etc\hosts`

2. Agregar: `127.0.0.1   local.clarovideo.com`

3. Guardar y reiniciar el navegador



**Solución Linux/Mac:**

```bash

echo "127.0.0.1   local.clarovideo.com" | sudo tee -a /etc/hosts

```

## 6. Tests Fallan con `Cannot find module`



**Síntoma:**

```

Cannot find module 'containers/MyContainer' from 'src/react/app.test.js'

```



**Causa:** Los imports sin ruta relativa (`'containers/...'`) dependen del resolve de Webpack, no de Jest.



**Solución:** Verificar `jest.config.js` — debería tener `moduleDirectories` o `modulePaths` que incluya `src/react`. Si no existe, agregar:

```javascript

```

## 🔧 Problemas de Setup



### Environment Issues

- **`npm start` falla sin error claro:** Verificar que `NODE_ENV=development` esté definido en el shell antes de ejecutar

- **Puerto en uso:** Si el puerto 3000 ya está ocupado, cambiar en `server/port.js` o usar `PORT=3001 npm start`

- **Variables de entorno faltantes:** Copiar `.env.example` → `.env` y completar todas las variables requeridas

- **`node_modules` corrupto o desactualizado:** Eliminar completamente y reinstalar: `Remove-Item -Recurse -Force node_modules; npm install`



### Dependency Problems

- **Conflictos de versión (`ERESOLVE`):** Usar `npm install --legacy-peer-deps` — React 17 tiene dependencias con requisitos de peerDependencies en conflicto

- **Paquetes no encontrados:** Verificar versión de Node.js (`>=8.10.0`, recomendada v18.x) con `node -v`

- **`react-scripts` no encontrado:** Este proyecto no usa CRA — usar `npm start` que invoca el script definido en `package.json` con Webpack

- **Módulos nativos:** Si hay errores de compilación de módulos nativos, instalar build tools: `npm install --global --production windows-build-tools`



### Configuration Errors

- **`NODE_PATH` indefinido:** Agregar `NODE_PATH=src` en `.env` para que los alias de importación funcionen correctamente

- **Webpack alias no resuelto:** Verificar `resolve.alias` en `config/webpack.common.js` — los alias deben incluir todos los paths usados en `src/react/`

- **Babel no transpila correctamente:** Verificar `babel.config.js` — debe incluir `@babel/preset-react`, `@babel/preset-env` y los plugins de optional chaining

- **PostCSS error:** Verificar versión de `postcss.config.js` — debe ser compatible con Webpack 5



### Permission Issues

- **HTTPS certificate error en desarrollo:** Si se usa `https`, generar certificado local con `mkcert` y configurar en `webpack.dev.js`

- **Permisos en scripts `.sh`:** En Windows usar Git Bash o WSL para ejecutar `scripts/*.sh`

- **Archivos de caché de Webpack bloqueados:** Eliminar `.cache/` y `dist/` antes de reintentar build

## 🏗️ Build y Compile



### Build Failures

- **`Error: Cannot find module 'webpack'`:** Ejecutar `npm install` — Webpack está en `devDependencies` y debe estar instalado localmente

- **Heap out of memory:** Aumentar memoria de Node.js: `set NODE_OPTIONS=--max_old_space_size=4096` antes de ejecutar `npm run build`

- **Build termina sin output:** Revisar `config/webpack.prod.js` — verificar que `output.path` y `entry` estén correctamente configurados

- **`SplitChunksPlugin` error:** Validar que `optimization.splitChunks` en `webpack.prod.js` no tenga configuración inválida



### Dependency Conflicts

- **`react-router` conflicto v3 vs v6:** El proyecto tiene ambas versiones coexistiendo — importar siempre de la versión correcta según el módulo. Código nuevo usa v6 (`react-router-dom` v6)

- **`redux` vs `react-redux` mismatch:** Verificar que las versiones en `package.json` sean compatibles entre sí

- **`sass` vs `node-sass`:** Este proyecto usa `sass` (Dart Sass 1.68.0) — no instalar `node-sass`



### Version Mismatches

- **Babel transform error en optional chaining:** Verificar que `@babel/plugin-proposal-optional-chaining` esté configurado en `babel.config.js`

- **PostCSS autoprefixer warnings:** Son advertencias, no errores — pueden ignorarse en desarrollo

- **`mini-css-extract-plugin` error:** Asegurar versión compatible con Webpack 5 en `package.json`



### Platform-specific Issues

- **Windows: paths con backslash:** Webpack normaliza paths — usar siempre `/` en imports de JS/SCSS

- **Windows: LESS compilation:** Verificar que `less-loader` y `less` estén instalados si se requiere compilar archivos `.less` legacy

- **macOS: certificado de desarrollo:** Seguir instrucciones en `DEV_SETUP.md` para trust del certificado local

## 🏃‍♂️ Runtime Issues



### Application Crashes

- **`Cannot read property of undefined` en Redux:** Verificar que el selector usa un valor inicial en el reducer correspondiente en `src/react/reducers/`

- **`Saga cancelled` inesperado:** Revisar el flujo en `src/react/appSaga.js` — puede haber un `takeLatest` cancelando peticiones anteriores

- **React `Maximum update depth exceeded`:** Detectar loops en `useEffect` — verificar dependencias del array `[]`

- **`window is not defined` en SSR:** Usar guards `typeof window !== 'undefined'` para código que accede a `window`



### Performance Problems

- **Re-renders excesivos:** Usar React DevTools Profiler para identificar componentes que re-renderizan sin cambios reales; asegurar memoización con `useSelector` y selectores Reselect

- **Bundle muy grande:** Ejecutar `npm run build` y revisar el output de tamaños de chunks — usar `webpack-bundle-analyzer` si está configurado

- **Imágenes lentas:** Verificar que `react-lazyload` esté activo para imágenes fuera del viewport



### Memory Leaks

- **`Warning: Can't perform a React state update on an unmounted component`:** Cancelar suscripciones y sagas al desmontar componentes con `useEffect` cleanup

- **Redux store creciendo ilimitadamente:** Revisar que los reducers limpien correctamente el estado con actions de reset

- **Event listeners no removidos:** Siempre retornar función de limpieza en `useEffect` que agrega listeners



### Network Issues

- **CORS en desarrollo:** El servidor Express en `server/index.js` maneja CORS — verificar `configureRouter.js` para los middlewares aplicados

- **Axios request falla silenciosamente:** Agregar interceptor global en `src/react/api/` para loguear errores en desarrollo

- **Timeout en peticiones a API:** Verificar config de timeout en las instancias de Axios en `src/react/api/`

## 🧪 Testing Problems



### Test Failures

- **`Cannot find module '...'`:** Alias de Webpack no resuelto en Jest — verificar `moduleNameMapper` en `jest.config.js`; debe mapear los mismos alias que `config/webpack.common.js`

- **`Enzyme adapter not configured`:** Falta configuración en `jest.setup.js` — asegurar que `setupFilesAfterFramework` en `jest.config.js` apunta a `jest.setup.js`

- **Snapshot fails inesperadamente:** El componente fue actualizado — ejecutar `npm test -- -u` para actualizar snapshots; revisar que el cambio sea intencional



### Flaky Tests

- **Tests asíncronos inconsistentes:** Usar `waitFor` de `@testing-library/react` o `act()` para esperar renders asíncronos

- **Sagas en tests:** Mockear las llamadas a API con `jest.mock()` en tests que involucren sagas

- **Orden de tests importa:** Tests con estado compartido pueden fallar dependiendo del orden — aislar estado con `beforeEach`/`afterEach`



### Test Environment Issues

- **`jsdom not found`:** Verificar `testEnvironment: 'jsdom'` en `jest.config.js`

- **`localStorage` no disponible:** jsdom provee `localStorage` — si falla, verificar que `testEnvironment` sea `jsdom` y no `node`

- **Tests lentos en CI:** Usar `--maxWorkers=2` en CI para evitar OOM; localmente usar `npm test -- --testPathPattern=<archivo>` para aislar



### Coverage Problems

- **Coverage report no genera:** Ejecutar explícitamente `npm test -- --coverage --watchAll=false`

- **Coverage bajo en módulos ESPN/HBO:** Agregar tests en `src/espn/` y `src/hbo/` — estos módulos tienen poca cobertura por ser integraciones externas

- **Archivos excluidos del coverage:** Revisar `coveragePathIgnorePatterns` en `jest.config.js`

## 🔄 Diagramas de Flujos de Procesos



### Flujo Principal 1: Ciclo Redux + Sagas



**Diagrama 3** documenta el flujo unidireccional completo de datos en la aplicación:



- Usuario interacóa → Container dispara `action.REQUEST`

- Redux Store → redux-saga intercepta con `takeLatest`

- Saga llama al API Service (Axios) → respuesta

- Saga dispara `action.SUCCESS` → Reducer actualiza estado

- `useSelector(Reselect selector)` → Component re-renderiza



**Fuentes:** `src/react/store.js`, `src/react/appSaga.js`, `src/react/api/`, `src/react/reducers/`



### Flujo Principal 2: Autenticación OAuth



**Diagrama 4** documenta el flujo de autenticación para todos los proveedores soportados:



- **Facebook OAuth:** `react-facebook-login` → token → UserAPI

- **Apple Sign-In:** `react-apple-login` → token → UserAPI

- **ESPN:** `src/espn/services.js` → token ESPN → UserAPI

- **HBO:** `src/hbo/services.js` → token HBO → UserAPI



Todos los proveedores convergen al mismo flujo Redux: `loginRequest` → Saga → `loginSuccess` → Cookie middleware → UserContext.

- `store.js`, `appSaga.js` → actualizar Diagramas 3 y 4 (flujos Redux/Auth)

- `config/webpack.common.js` → actualizar Diagrama 5 (Build Pipeline)
