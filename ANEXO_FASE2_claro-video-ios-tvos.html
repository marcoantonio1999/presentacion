### SEC-01 — Token NewRelic Hardcodeado ❌ CRÍTICO

| Campo | Detalle |
|-------|---------|
| **ID** | SEC-01 |
| **Severidad** | 🔴 CRÍTICO |
| **OWASP** | A02:2021 — Cryptographic Failures / Sensitive Data Exposure |
| **MASVS** | MSTG-STORAGE-14: No almacenar secretos en código fuente |
| **Archivo** | `ClarovideoTVOS/SupportingFiles/AppDelegate.swift:25` |
| **Estado** | ABIERTO — en producción |

**Evidencia directa:**
```swift
// AppDelegate.swift, línea 25
NewRelic.start(withApplicationToken:"AAe55f2350abf5d377ed075c02057b1e293e38245a-NRMA")
```

**Riesgo:**
- El token expuesto permite a cualquier actor malicioso **reportar datos falsos** a la cuenta de NewRelic del proyecto, saturar el plan de monitoreo y generar costos no autorizados.
- Un atacante puede **suplantar el agente de telemetría** e inyectar datos de performance falsificados.
- El token otorga acceso de escritura a trazas de error, logs y métricas — información que podría revelar la topología interna de servicios backend.
- Al estar versionado en Git, el token **persiste en el historial** aunque sea eliminado del código (requiere revocación + git-filter-branch).

**CVSS v3.1 (estimado):** AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:M/A:L → **8.2 ALTO**

### SEC-02 — Firebase API Key y CLIENT_ID Hardcodeados ❌ CRÍTICO

| Campo | Detalle |
|-------|---------|
| **ID** | SEC-02 |
| **Severidad** | 🔴 CRÍTICO |
| **OWASP** | A02:2021 — Sensitive Data Exposure |
| **MASVS** | MSTG-STORAGE-14 |
| **Archivo** | `ClarovideoTVOS/SupportingFiles/GoogleService-Info.plist` |
| **Estado** | ABIERTO — versionado en repositorio |

**Evidencia directa:**
```xml
<key>API_KEY</key>
<string>AIzaSyClmLcwK4oMREJvSCN5LhaRQqtHxpmW4ss</string>
<key>CLIENT_ID</key>
<string>639595339410-4ulbaseh3afjmtav7sb94l5tbe2cgtfk.apps.googleusercontent.com</string>
<key>GOOGLE_APP_ID</key>
<string>1:639595339410:ios:21039f8b49c600fb</string>
<key>PROJECT_ID</key>
<string>claro-video-c48c7</string>
```

**Riesgo:**
- La `API_KEY` de Firebase expuesta permite a atacantes **abusar de servicios de Firebase** (Firestore, Remote Config, FCM) en nombre del proyecto, incurriendo en costos y potencialmente accediendo a datos de usuarios.
- El `CLIENT_ID` y `GOOGLE_APP_ID` permiten ingeniería inversa de la infraestructura del proyecto.
- Aunque Google aplica restricciones de dominio, la API key puede ser usada en ataques de **quota exhaustion** o acceso a endpoints sin restricción de paquete.

**Nota de contexto:** Google recomienda que `GoogleService-Info.plist` sea incluido en el bundle de la app (es legítimo en runtime), pero el riesgo mayor reside en la falta de **restricciones de API** en Google Cloud Console. La mitigación primaria es restringir la API key al bundle ID `com.dla.ClaroVideo` en Google Cloud Console.

**CVSS v3.1 (estimado):** AV:N/AC:L/PR:N/UI:N/S:U/C:M/I:M/A:L → **6.5 MEDIO-ALTO**

### SEC-03 — App Transport Security Completamente Deshabilitado ⚠️ ALTO

| Campo | Detalle |
|-------|---------|
| **ID** | SEC-03 |
| **Severidad** | 🟠 ALTO |
| **OWASP** | A05:2021 — Security Misconfiguration |
| **MASVS** | MSTG-NETWORK-1: Toda comunicación usa TLS |
| **Archivo** | `ClarovideoTVOS/SupportingFiles/ClarovideoTVOS-Info.plist:53-56` |
| **Estado** | ABIERTO — en producción |

**Evidencia directa:**
```xml
<key>NSAppTransportSecurity</key>
<dict>
    <key>NSAllowsArbitraryLoads</key>
    <true/>
    <!-- NSExceptionDomains también presentes — confirma coexistencia -->
</dict>
```

**Riesgo:**
- `NSAllowsArbitraryLoads = true` **desactiva ATS globalmente**, permitiendo conexiones HTTP sin cifrar a **cualquier** dominio, incluyendo streams de video, autenticación y metadata.
- Exposición a ataques **Man-in-the-Middle (MitM)** en redes Wi-Fi no confiables (hotspots de hotel, aeropuerto).
- Credenciales de usuario (tokens de sesión, refresh tokens) pueden transmitirse en texto plano.
- Apple rechaza apps con este flag en App Store Review sin justificación válida — riesgo de rechazo en próxima submission.

**CVSS v3.1 (estimado):** AV:N/AC:H/PR:N/UI:R/S:U/C:H/I:H/A:N → **6.8 MEDIO**

### SEC-05 — Force Casts en Código de Producción ⚠️ MEDIO

| Campo | Detalle |
|-------|---------|
| **ID** | SEC-05 |
| **Severidad** | 🟡 MEDIO |
| **OWASP** | A05:2021 / Reliability |
| **SonarQube** | `SwiftLint:force_cast` — MAJOR severity, BUG type |
| **Instancias** | 12 ocurrencias (evidencia: `sonar_issues.csv`) |
| **Estado** | En código fuente activo |

**Riesgo:**
- Un force cast (`as!`) que falla en runtime causa un **crash irrecuperable** (`EXC_BAD_INSTRUCTION`).
- Si el crash ocurre durante una transacción de pago o manejo de datos de usuario, puede resultar en pérdida de datos o estado inconsistente.
- Clasificado como BUG por SonarQube — impacto en disponibilidad (CWE-617: Reachable Assertion).
