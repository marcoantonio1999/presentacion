### SEC-004: Manejo Seguro de Tokens en roRegistry

**Severidad:** 🟡 MEDIA | **Esfuerzo:** Medio (2 días)

```brightscript
' TokenManager.brs — Patrón recomendado
' Ubicación: source/GHRDK/TokenManager.brs

' Principios aplicados:
' 1. Nunca loggear tokens
' 2. Limpiar en logout
' 3. Verificar expiración antes de usar
' 4. Usar sección privada de Registry

function SaveUserToken(token as String, expiresAt as Integer) as Void
  registry = CreateObject("roRegistrySection", "auth")
  registry.Write("access_token", token)
  registry.Write("expires_at", Str(expiresAt))
  registry.Flush()
end function

function GetUserToken() as String
  registry = CreateObject("roRegistrySection", "auth")
  token = registry.Read("access_token")
  expiresAt = Val(registry.Read("expires_at"))
  
  now = CreateObject("roDateTime").AsSeconds()
  if now >= expiresAt
    ' Token expirado — limpiar y retornar vacío
    ClearUserToken()
    return ""
  end if
  
  return token
end function

function ClearUserToken() as Void
  registry = CreateObject("roRegistrySection", "auth")
  registry.Delete("access_token")
  registry.Delete("expires_at")
  registry.Flush()
end function
```

- **Patrón Principal:** Modelo Vista Controlador (MVC) adaptado a Roku

### Componentes Principales
- **Presentation Layer:** 20+ Pages (Home, Login, Player, Live, Search, etc.)
- **Framework Layer:** 35 componentes GHRDK (Buttons, Lists, Video, Input, etc.)
- **Business Logic:** 4 componentes lógica (JumpPages, DeepLinking, JumpStart, Router)
- **API Layer:** 25+ servicios REST (User, Media, Search, Analytics, etc.)
