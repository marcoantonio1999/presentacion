## Deuda de Arquitectura

### 1. Falta de Modularización Gradle

**Problema:** Los 18 módulos de features (`analytics/`, `player/`, `search/`, etc.) existen como **directorios de paquetes** dentro del único módulo Gradle `:app`, no como módulos independientes.

| Consecuencia | Impacto |
|---|---|
| Tiempos de compilación largos | Todo se compila junto siempre |
| Imposibilidad de tests unitarios aislados | No se puede mockear módulos |
| Acoplamiento implícito entre features | Cambios en `core/` pueden romper `player/` |
| Riesgo de regresión en PRs | — |

**Fuente:** `app/build.gradle` (único módulo), `settings.gradle` (sin `include ':player'` u otros)

### 2. Coexistencia RxJava3 + Coroutines (Dual Async)

**Problema:** El módulo `tv/` (EPG) usa RxJava3 para observables mientras el resto de la app usa Kotlin Coroutines/Flow. Esto introduce:

| Consecuencia | Impacto |
|---|---|
| Dos paradigmas de manejo de errores | Mayor curva de aprendizaje, bugs de integración |
| Overhead de conversión `rx → flow` en boundaries | Performance y code smell |
| Dependencias adicionales sin beneficio nuevo | +300KB APK aprox. |

**Fuente:** `app/build.gradle` — `io.reactivex.rxjava3:rxjava:3.0.6` + `io.reactivex.rxjava3:rxandroid:3.0.0`

### 3. Coexistencia Leanback + Jetpack Compose (Dual UI)

**Problema:** La app usa Leanback (View-based, para TV) y Jetpack Compose en distintas partes.

**Fuente:** `app/build.gradle` — `androidx.leanback:leanback:1.0.0` + compose UI 1.1.1
