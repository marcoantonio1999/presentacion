## 1. Introducción al Estándar ISO/IEC 5055

ISO/IEC 5055 (también conocido como CISQ — Consortium for IT Software Quality) define métricas para medir la calidad de software a través de cuatro características estructurales principales:

1. **Seguridad** — Capacidad del software de resistir ataques y proteger datos
2. **Confiabilidad** — Capacidad del software de funcionar correctamente bajo las condiciones esperadas
3. **Mantenibilidad** — Facilidad para modificar y extender el software
4. **Eficiencia de Performance** — Capacidad del software de responder y usar recursos de manera óptima

El estándar mide estas características contando el número y severidad de "debilidades" en el código que pueden comprometer cada una de las propiedades.

## 4. Análisis Detallado de los Bugs más Críticos

### 4.1 `css:S4670` — 71 ocurrencias CRITICAL (Selector CSS desconocido)

- **Archivos:** `src/css/bootstrap-claro-web.css`, `src/less/vph5-player.less`
- **Mensaje SonarQube:** "Unexpected unknown type selector `videoph5`"
- **Impacto:** El selector CSS `videoph5` es un elemento HTML personalizado del reproductor VPH5 heredado. Genera 71 advertencias CRITICAL en el archivo de estilos global más importante del proyecto. Puede afectar el rendering en navegadores modernos.
- **Causa raíz:** CSS heredado del reproductor `vph5` que utiliza selectores de tipo personalizado en lugar de clases o IDs convencionales.

### 4.2 `javascript:S6756` — 174 ocurrencias MAJOR (Race condition en setState)

- **Archivos:** `src/react/components/terms/Terms.js`, `CVVForm/index.js`, `CreditCardForm/index.js`, y otros
- **Mensaje SonarQube:** "Use callback in setState when referencing the previous state."
- **Impacto:** El patrón `setState(this.state.x + 1)` en lugar del correcto `setState(prev => ({x: prev.x + 1}))`. En React StrictMode o durante renders síncronos rápidos, este patrón produce bugs de estado obsoleto que son intermitentes y difíciles de depurar.
- **Causa raíz:** Patrón incorrecto extendido en múltiples componentes, probablemente adoptado desde patrones de React 15/16 sin actualizar.

### 4.3 `javascript:S6440` — 29 ocurrencias MAJOR (Violación de Reglas de Hooks)

- **Archivos:** `AlertModal/index.js` (3 bugs: L20, L21, L51), `CvvFormV2.js`, y otros
- **Mensaje SonarQube:** "React Hook `useSelector`/`useEffect` is called conditionally."
- **Impacto:** Violación directa de las Reglas de Hooks de React. Causa crashes de React en producción cuando el orden de hooks cambia entre renders. Afecta a componentes críticos en los flujos de alerta y pago.
- **Causa raíz:** Hooks declarados dentro de bloques `if` o después de un early `return null`.

### 4.4 `javascript:S930` — Múltiples CRITICAL (Argumentos incorrectos en funciones)

- **Archivos:** `ScheduleCell.js`, `PlanSelectorCarousel/index.js`, `RecordingOptions.js`, `PlanSelect.js`
- **Mensaje SonarQube:** "This function expects N arguments, but M were provided."
- **Impacto:** Errores silenciosos o crashes en runtime al invocar funciones con el número incorrecto de argumentos. Especialmente riesgoso en el módulo de pagos y en el listado de planes.
- **Causa raíz:** Refactoring de firmas de función sin actualizar todos los call sites correspondientes.

### 4.5 `javascript:S6523` — 22 ocurrencias MAJOR (TypeError en runtime)

- **Archivos:** `nav/actions.js`, `AddOnCheckoutV2.js`, `RegisterV2/index.js`, `livePlayer.utils.js`
- **Mensaje SonarQube:** "Unsafe usage of optional chaining. If it short-circuits with `undefined` the evaluation will throw TypeError."
- **Impacto:** Uso de `?.()` en expresiones donde el cortocircuito con `undefined` lanzaría `TypeError`. Crash potencial en flujos de pago, registro y reproducción en vivo.

### 4.6 `css:S4649` — 428 ocurrencias MAJOR (Familia de fuente incompleta)

- **Archivos:** Dispersos en múltiples archivos SCSS/CSS a lo largo de `src/react/`
- **Mensaje SonarQube:** "Unexpected missing generic font family"
- **Impacto:** Declaraciones `font-family` sin familia genérica de respaldo (ej: `sans-serif`). Puede causar que los navegadores no encuentren la fuente y usen la del sistema operativo en lugar del diseño previsto.

### R-01: Crashes en producción — Hooks condicionales de React

**Severidad:** 🔴 CRÍTICO
**Fuente Sonar:** `javascript:S6440` — 29 instancias OPEN

**Descripción:** Los hooks de React (`useSelector`, `useEffect`) son llamados condicionalmente o después de un early return. Esto viola directamente las Reglas de Hooks de React. En React 17, puede causar crashes silenciosos o errores visibles en flujos específicos de usuario.

### SEC-04: Corregir Hooks Condicionales — `javascript:S6440` (29 instancias)

**Prioridad:** 🔴 CRÍTICA
**Esfuerzo estimado:** ~5 horas
**OWASP:** A04:2021 — Diseño Inseguro (fallo de seguridad de la UI en crashes)
**CWE:** CWE-248 — Uncaught Exception
**Archivo de origen:** `sonar/sonar_bugs.csv` — regla S6440, 29 instancias OPEN

**Contexto:**
Llamar a hooks de React (`useSelector`, `useEffect`, `useState`) condicionalmente viola las Reglas de Hooks de React y causa crashes de la aplicación durante el render. Cuando estos crashes ocurren en componentes de autenticación o de pago, el comportamiento resultante puede ser inseguro (mostrar UI de estado intermedio).

**Archivos más críticos a corregir primero:**
1. `src/react/components/AlertModal/index.js` — Líneas L20, L21, L51
2. `src/react/components/Payment/CvvFormV2.js`

**Patrón incorrecto (a corregir):**

```javascript
// ❌ INCORRECTO — Hook llamado condicionalmente
const MyComponent = ({ isLoggedIn }) => {
  if (!isLoggedIn) {
    return null; // early return ANTES de los hooks
  }
  const data = useSelector(mySelector); // Hook después de return condicional
  // ...
};
```

**Patrón correcto:**

```javascript
// ✅ CORRECTO — Todos los hooks al inicio, antes de cualquier return condicional
const MyComponent = ({ isLoggedIn }) => {
  const data = useSelector(mySelector); // Hook SIEMPRE se llama

  if (!isLoggedIn) {
    return null; // El early return va DESPUÉS de todos los hooks
  }
  // ...
};
```

**Validación post-cambio:**
- [ ] SonarQube no reporta nuevas instancias de S6440
- [ ] Los tests de `AlertModal` y `CvvFormV2` pasan
- [ ] La aplicación no genera warnings de React sobre el orden de hooks en la consola
