## 📋 Resumen Arquitectónico

### Stack Tecnológico

**Tecnologías Principales Detectadas:**
- **Lenguaje:** Kotlin 2.1.0 (primary), Java (legacy)
- **Arquitectura:** MVVM + Clean Architecture (5 capas)
- **UI:** Jetpack Compose 1.6.8 + XML layouts
- **DI:** Hilt (Dagger 2.44)
- **Async:** Coroutines + Flow + LiveData
- **Streaming:** Media3 (ExoPlayer 1.3.1) con DRM
- **Network:** Retrofit + OkHttp
- **Database:** Room (SQLite)
- **Testing:** JUnit 4, Mockito, Espresso
- **Build:** Gradle (AGP 8.10.0)

### Patrones Arquitectónicos Principales

**MVVM + Clean Architecture:**
- 5 capas: Presentation → ViewModel → Domain → Data → Core
- Separación clara de responsabilidades
- Inversión de control mediante Hilt
- Programación reactiva con Flow/StateFlow

### Principios de Diseño

1. **Single Responsibility** - Una responsabilidad por clase
2. **Dependency Inversion** - Inyectar dependencias
3. **Open/Closed** - Abierto para extensión
4. **Interface Segregation** - Interfaces específicas
5. **Liskov Substitution** - Subtypes intercambiables
6. **DRY** - Evitar repetición
7. **Composition over Inheritance** - Preferir composición
8. **Testability First** - Diseño orientado a tests

## 🏗️ Arquitectura y Diseño

### Principios SOLID

**S - Single Responsibility:**
```kotlin
// ✅ BUENO - Responsabilidad única
class VideoRepository(
    private val api: ApiService  // Solo acceso a datos
)

// ❌ MALO - Múltiples responsabilidades
class VideoService(
    private val api: ApiService
) {
    fun getVideo() { /* API */ }
    fun renderUI() { /* UI rendering */ }
    fun logAnalytics() { /* Analytics */ }
}
```

**O - Open/Closed:**
```kotlin
// ✅ BUENO - Abierto para extensión, cerrado para modificación
sealed class UiState {
    object Loading : UiState()
    data class Success(val data: Any) : UiState()
    data class Error(val exception: Exception) : UiState()
}

// Añadir nuevo estado sin modificar código existente
data class Empty(val message: String) : UiState()
```

**L - Liskov Substitution:**
```kotlin
// ✅ BUENO - Subtypes pueden substituir al tipo base
interface DataSource {
    suspend fun getData(): List<Video>
}

class RemoteDataSource : DataSource {
    override suspend fun getData() = api.getVideos()
}

class LocalDataSource : DataSource {
    override suspend fun getData() = db.getVideos()
}
```

**I - Interface Segregation:**
```kotlin
// ✅ BUENO - Interfaces específicas
interface VideoLoader {
    suspend fun load(id: String): Video
}

interface VideoPlayer {
    fun play(video: Video)
}

// ❌ MALO - Interface muy grande
interface VideoService {
    suspend fun load(id: String): Video
    fun play(video: Video)
    fun pause()
    fun stop()
}
```

**D - Dependency Inversion:**
```kotlin
// ✅ BUENO - Depender de abstracciones
class VideoViewModel @Inject constructor(
    private val repository: VideoRepository  // Interface
)

// ❌ MALO - Depender de implementación concreta
class VideoViewModel(
    private val repository: VideoRepositoryImpl
)
```

```kotlin
// Interface (Domain)
interface VideoRepository {
    suspend fun getVideo(id: String): Video
    fun watchVideos(): Flow<List<Video>>
}

// Implementación (Data)
class VideoRepositoryImpl @Inject constructor(
    private val remoteSource: VideoRemoteDataSource,
    private val localSource: VideoLocalDataSource
) : VideoRepository {
    override suspend fun getVideo(id: String): Video {
        return try {
            remoteSource.getVideo(id).also { video ->
                localSource.save(video)
            }
        } catch (e: Exception) {
            localSource.getVideo(id) ?: throw e
        }
    }
}
```


```
6. Signing: Digital signature
7. Alignment: zipalign para optimización
```

## 🐞 Your First Feature (Step-by-Step)

### Scenario: Add a "Logout" button to settings

**Step 1: Understand existing code (30 min)**
```
Find: SettingsViewModel.kt
Find: SettingsFragment.kt
Find: AuthService.kt

Trace: Button Click → ViewModel → UseCase → Repository → Service
```

**Step 2: Create UseCase (5 min)**
```kotlin
// File: domain/usecases/LogoutUseCase.kt
class LogoutUseCase @Inject constructor(
    private val authRepository: AuthRepository
) {
    suspend operator fun invoke() {
        authRepository.logout()
    }
}
```

**Step 3: Update ViewModel (5 min)**
```kotlin
// File: presentation/viewmodels/SettingsViewModel.kt
@HiltViewModel
class SettingsViewModel @Inject constructor(
    private val logoutUseCase: LogoutUseCase
) : ViewModel() {
    fun logout() {
        viewModelScope.launch {
            try {
                logoutUseCase()
                // Navigate to login
            } catch (e: Exception) {
                // Show error
            }
        }
    }
}
```

**Step 4: Update UI (5 min)**
```kotlin
// File: presentation/composables/SettingsScreen.kt
@Composable
fun SettingsScreen(viewModel: SettingsViewModel) {
    Button(onClick = { viewModel.logout() }) {
        Text("Logout")
    }
}
```

**Step 5: Write Tests (15 min)**
```kotlin
// File: src/test/.../SettingsViewModelTest.kt
@Test
fun logout_shouldCallUseCase() = runTest {
    // Given
    coEvery { logoutUseCase() } just runs
    
    // When
    viewModel.logout()
    advanceUntilIdle()
    
    // Then
    coVerify { logoutUseCase() }
}
```

**Step 6: Commit & Push (5 min)**
```bash
git add .
git commit -m "feat: Add logout button to settings

- Implement LogoutUseCase
- Update SettingsViewModel with logout logic
- Add logout button to SettingsScreen
- Add unit tests

Closes #789"

git push origin feature/add-logout-button
```

## 🐛 Debugging

### 1. Configurar Debug Mode

Activar modo de debugging en Android Studio:

```bash
Run > Debug 'app' (Shift+F9)
```

**Pasos:**
1. Abre tu proyecto en Android Studio
2. **Run → Debug 'app'** (o presiona `Shift+F9` en Windows/Linux, `Ctrl+R` en macOS)
3. Elige emulador/dispositivo
4. App se ejecutará en modo debug
5. Se pausará automáticamente en breakpoints

**Breakpoints:**
- **Agregar:** Click en número de línea a la izquierda del código
- **Remover:** Click nuevamente en el punto rojo
- **Breakpoint condicional:** Right-click → "Condition" → enter condición

### 2. Logs útiles

#### Ver Logs en Logcat

```
View > Tool Windows > Logcat (Alt+6)
```

**Filtrar por tag:**
```bash
adb logcat | grep "dla.android"
```

**Niveles de log:**
```bash
adb logcat *:V    # Verbose (todas)
adb logcat *:D    # Debug
adb logcat *:I    # Info
adb logcat *:W    # Warning
adb logcat *:E    # Error
```

**Limpiar logs:**
```bash
adb logcat -c
```

#### Debugging con Breakpoints

1. **Click en número de línea** donde quieres pausar (left margin)
2. **Ejecutar en Debug** (Shift+F9)
3. **Navegar a ese código** (interacción en app)
4. **Se pausa automáticamente** en breakpoint
5. **Inspecciona variables** en Debug pane (abajo de pantalla)

**Atajos de navegación:**
- `F10` = **Step Over** (siguiente línea, sin entrar en funciones)
- `F11` = **Step Into** (entrar dentro de función)
- `Shift+F11` = **Step Out** (salir de función actual)
- `F8` = **Resume** (continuar ejecutando hasta próximo breakpoint)
- `Ctrl+F8` = **Mute breakpoints** (deshabilitar todos temporalmente)

**Variables & Watch:**
- Ver variables locales: Variables pane (izquierda)
- Inspeccionar variable: Hover mouse sobre variable en código
- Watch expression: Right-click variable → Add to Watches

## 🆘 Errores Comunes

### Error: "Java not found"
```
❌ Error: could not find java
✅ Solución: Instalar JDK 17, configurar JAVA_HOME
```

### Error: "Build failed: build-tools not found"
```
❌ Error: android build-tools not found
✅ Solución:
   1. Android Studio → SDK Manager
   2. Instalar Build-Tools 34.0.0+
   3. ./gradlew clean
```

### Error: "Gradle sync failed"
```
❌ Error: Could not determine Gradle version
✅ Solución:
   1. File → Sync Now
   2. O en terminal: ./gradlew --refresh-dependencies
   3. ./gradlew clean
   4. Reiniciar Android Studio
```

### Error: "Emulator: permission denied"
```
❌ Error: permission denied when running emulator
✅ Solución (Linux):
   chmod +x ~/Android/Sdk/emulator/emulator
```

### Error: "APK installation failed"
```
❌ Error: cmd: Can't find service manager
✅ Solución:
   1. Desconectar/reconectar dispositivo
   2. Reinstalar drivers adb
   3. Probar con: adb install -r app.apk
```

### Error: "App crashes on launch"
```
❌ App abre pero cierra inmediatamente
✅ Solución:
   1. Ver logs: adb logcat -s "TAG"
   2. Verificar AndroidManifest.xml
   3. Revisar config en local.properties
   4. Clean + rebuild: ./gradlew clean assembleDebug
```

### Error: "Tests fail"
```
❌ Test failed: assertion error
✅ Solución:
   1. Leer mensaje de error completo
   2. Verificar Mockito setup
   3. Revisar test configuration
   4. Ejecutar un test a la vez: ./gradlew testDebug --tests "TestClass"
```

## 🔧 Problemas de Setup

### Environment Issues

**JAVA_HOME no configurado:**
```bash
# Windows
setx JAVA_HOME "%JAVA_HOME_PATH%"

# Mac/Linux
export JAVA_HOME=$(/usr/libexec/java_home -v 17)

# Android Studio
Settings > Build > Gradle > Gradle JDK > jbr-17
```

### Dependency Problems

**Dependencias corruptas:**
```bash
# Clean Gradle cache
rm -rf ~/.gradle
./gradlew clean build
```

**Versiones incompatibles:**
- Verificar `gradle/wrapper/gradle-wrapper.properties`
- Sincronizar `build.gradle` con versiones correctas

### Configuration Errors

**Android SDK no encontrado:**
```bash
# Linux/Mac
sdkmanager --licenses
export ANDROID_HOME=$HOME/Android/Sdk

# Windows
set ANDROID_HOME=%ANDROID_SDK_PATH%
```

### Permission Issues

**Permisos insuficientes:**
- Linux: `chmod +x gradlew`
- Dispositivo: Habilitar "Instalar desde fuentes desconocidas"
- Carpeta: Asegurar acceso de lectura/escritura

```gradle
// En build.gradle
configurations.all {

./gradlew app:dependencies
```

## 🏃‍♂️ Runtime Issues

### Application Crashes

**Crash al iniciar:**
```bash
# Ver logcat
adb logcat | grep FATAL

# Stack trace
adb logcat AndroidRuntime:E *:S
```

**Causas comunes:**
- NullPointerException en onCreate
- Recurso/archivo no encontrado
- ProGuard/R8 ofuscación incorrecta

### Performance Problems

**App lenta/laggy:**
- Instrumentation profiler
- Memory monitor (Androidkitkat+)
- Use callstack sampling

**Soluciones:**
- Optimizar layouts (reducir nested views)
- Lazy loading de imágenes
- Caché de datos frecuentes

### Memory Leaks

**OutOfMemoryError:**
```bash
# Hprof dump
adb shell am dumpheap <pid> /data/anr/heap.hprof

# Analizar con Android Studio
Profiler > Memory > Export HPROF
```

### Network Issues

**Timeouts/Connection errors:**
- Verificar conectividad (ping)
- Certificado SSL válido
- Proxy/VPN configurado correctamente
- Reintentos con backoff exponencial

## 🧪 Testing Problems

### Test Failures

**Tests fallan localmente pero pasan en CI:**
- Diferencia en timezone/locale
- Datos de prueba no sincronizados
- Ejecución en diferente orden

**Soluciones:**
```bash
# Ejecutar en orden específico
./gradlew test -i

# Ver stderr/stdout
./gradlew test -info
```

### Flaky Tests

**Tests que pasan/fallan intermitentemente:**
- Condiciones de carrera (async code)
- Timeouts insuficientes
- State no limpiado entre tests

**Fix:**
```kotlin
// Aumentar timeout
@Test(timeout = 5000L)
fun test() { ... }

// Usar @Before/@After
@Before
fun setUp() { /* Reset state */ }
```

### Test Environment Issues

**Mock framework no funciona:**
- Verificar import correcto (mockk vs mockito)
- Asegurar dependencies en test build.gradle
- Limpiar caché del IDE

### Coverage Problems

**Coverage no se genera:**
```bash
./gradlew testDebugUnitTestCoverage

# Ver reporte
open build/reports/coverage/debug/index.html
```

## 🛡️ Aspectos de Seguridad

### 1. Puntos de Autenticación

**OAuth 2.0 Flow**
```
1. User enters credentials
    ↓
2. App sends: POST /auth/login {email, password}
    ↓
3. Server validates
    ↓
4. Returns: JWT token + refresh token
    ↓
5. App stores en Android KeyStore (encriptado)
    ↓
6. Todos los requests posteriores incluyen: Authorization: Bearer {token}
```

**Token Management**
- Almacenamiento: Android KeyStore (encriptación por hardware si disponible)
- Refresh: Automático mediante Retrofit Interceptor

```python
# Repository coordina múltiples fuentes de datos
class VideoRepositoryImpl(
    private val remoteDataSource: RemoteVideoDataSource,
    private val localDataSource: LocalVideoDataSource
) : VideoRepository {
    override suspend fun getVideo(videoId: String): Video {
        // Primero intenta cache local
        val cached = localDataSource.getVideo(videoId)
        if (cached != null && !expired(cached)) {
            return cached
        }
        
        // Si no existe, fetcha desde API
        val fresh = remoteDataSource.getVideo(videoId)
        localDataSource.saveVideo(fresh)
        return fresh
    }
}
```

### Error Handling Strategy

#### Global Error Handling

```python
# Repository maneja errores y convierte a domain exceptions
override suspend fun getVideo(videoId: String): Video {
    return try {
        remoteDataSource.getVideo(videoId)
    } catch (e: HttpException) {
        when (e.code()) {
            401 -> throw AuthenticationException("Token expired")
            404 -> throw VideoNotFoundException("Video not found")
            403 -> throw LicenseException("DRM license invalid")
            else -> throw NetworkException("Network error: ${e.message}")
        }
    } catch (e: IOException) {
        throw NetworkException("Network unreachable")
    }
}

# ViewModel convierte a UI state
viewModelScope.launch {
    try {
        val video = repository.getVideo(videoId)
        _uiState.value = UiState.Success(video)
    } catch (e: VideoNotFoundException) {
        _uiState.value = UiState.Error("El video no existe")
    } catch (e: LicenseException) {
        _uiState.value = UiState.Error("Licencia DRM inválida")
    } catch (e: Exception) {
        _uiState.value = UiState.Error("Error inesperado: ${e.message}")
    }
}
```
