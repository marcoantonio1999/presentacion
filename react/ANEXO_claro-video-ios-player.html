Hotspots confirmados

  • VOD, SUBSCRIPTION y LIVESTREAMING concentran la mayor carga funcional del paquete y sostienen el camino critico del reproductor.
  • El alcance de 6 dominios es intencional: este paquete representa el SDK/player, no el mapa funcional completo de la app iOS principal.
  • TIMESHIFT, RECORDED y MUSIC son mas pequenos en volumen, pero dependen del mismo stack compartido de player, DRM y estado global.

Riesgos operativos

  • MUSIC-001: ProviderAMCOMusic.swift deja hardcodeada la URL del certificado FairPlay de musica.
  • VOD-003: BitmovinConstants.swift mantiene licencias de Bitmovin Player y Analytics hardcodeadas.
  • VOD-004 y TIMESHIFT-001: SharedInfo.swift mezcla estado global mutable y el flag isNoInLiveWithTimeShift.
  • LIVESTREAMING-001 y SUBSCRIPTION-001: enums y contratos con semantica ambigua como OptionLive y el typo purchaseSeasson.

Siguiente movimiento

  • Priorizar FASE 2 sobre VOD, LIVESTREAMING y SUBSCRIPTION, porque concentran playback, DRM y monetizacion.
  • Externalizar certificados y licencias, y reducir el estado global mutable antes de seguir ampliando features del player.
  • Usar VALIDATION_REPORT.md como consolidado funcional inicial y la seccion 15 del repo_graph.graphml como respaldo estructural.
  • Documentar dependencias que hoy quedan fuera del repo, como Apple IAP e infraestructura backend de music/timeshift/NPVR.

# Quick Start Guide - cv-ios-player

## 🚀 Introducción
`cv-ios-player` es un SDK Swift de reproducción de video para iOS/tvOS distribuido por Swift Package Manager, con módulos `CVPlayer`, `CVPlayerTVOS` y `CVPlayerCommon`.

## 📋 Prerrequisitos
- [ ] macOS con Xcode 14.3+
- [ ] Acceso al repositorio
- [ ] Simulador o dispositivo iOS 16+/tvOS 16+
- [ ] Credenciales FairPlay si se usa contenido protegido

## ⚙️ Configuración del Entorno
### 1. Instalar Herramienta Principal
- Instalar/actualizar Xcode 14.3 o superior.

### 2. Configurar SDK/Framework
- Abrir `VideoPlayerSwift.xcodeproj` y resolver dependencias SPM.

### 3. Configurar Dependencias
- Verificar resolución de dependencias en `Package.swift`.

## 📁 Clonar y Configurar Proyecto
### 1. Clonar el Repositorio
```bash
git clone <url-del-repo>
cd claro-video-ios-player-main
```

### 2. Configurar Archivos Secrets
- Configurar secretos en GitHub Actions para CI cuando aplique (`TF_GITHUB_TOKEN`).

### 3. Configurar Servicios Externos
- Preparar credenciales/licencias FairPlay para pruebas DRM.

## 🔨 Build del Proyecto
### 1. Sincronizar Dependencias
- Abrir el proyecto en Xcode para resolver SPM automáticamente.

### 2. Build Debug/Development
```bash
xcodebuild -scheme VideoPlayerSwift-Universal -configuration Debug build
```
**⏱️ Tiempo esperado:** 3-10 minutos

## 📱 Ejecutar la Aplicación
### 1. Run Configuration
- Seleccionar esquema `TestApp` en Xcode.

### 2. Ejecutar App
- Ejecutar en simulador/dispositivo compatible.

### 3. Verificar Instalación
- ✅ Compila `import CVPlayer` en target iOS
- ✅ Compila `import CVPlayerTVOS` en target tvOS
- ✅ Se puede instanciar `getVideoPlayerViewController()`

## 🧪 Ejecutar Tests
### Unit Tests
- `Tests/cv-ios-playerTests/VideoPlayerSwiftTests/`
- `PlayerTVTests/PlayerTVTests.swift`

### Integration Tests
- Validación manual en `TestApp`.
**🎯 Target de cobertura:** ≥ 80%

## 🐛 Debugging
### 1. Configurar Debug Mode
- Usar esquema `TestApp` en modo `Debug`.

### 2. Logs útiles
- Revisar callbacks en `Sources/CVPlayerCommon/Observers/`.

### 3. Herramienta de Debugging Específica
- Xcode Instruments (`Time Profiler`, `Leaks`).

## 🔧 Troubleshooting
### Errores Comunes
#### 1. Error Típico 1
- `ld: framework not found`: verificar producto enlazado (`CVPlayer`/`CVPlayerTVOS`).

#### 2. Error Típico 2
- Fallo de resolución SPM: `File > Packages > Reset Package Caches`.

### Performance Issues
- Evitar observers redundantes y trabajo pesado en hilo principal.
