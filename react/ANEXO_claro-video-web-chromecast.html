# Architecture Guide - cv-chromecast

> Guia tecnica de la arquitectura del Custom Receiver de Chromecast para ClaroVideo.
> **Archivos de evidencia:** `src/js/player.js` (2431 lineas), `src/js/app.js`, `src/js/constants.js`, `src/js/stringutils.js`, `webpack.config.js`

---

## Resumen Arquitectonico

### Stack Tecnologico

| Capa | Tecnologia | Version | Rol |
|------|-----------|---------|-----|
| Lenguaje | JavaScript ES6 | ES6 | Logica del receiver |
| Transpiler | Babel | 6.x | ES6 -> ES5 (Chrome 38 compat) |
| Bundler | Webpack | 1.x (^1.13.2) | Bundle unico para deploy |
| Cast Runtime | Google Cast SDK v3 | CDN | Receiver framework |
| Media Player | Shaka Player | 2.x (CDN) | DASH/DRM playback |
| CSS | PostCSS cssnext | --- | Estilos con autoprefixer |
| Target | Chrome 38+ | --- | Chromecast Gen 1-3 |

**Evidencia:** `package.json` (versiones), `src/index.html` (CDN scripts), `webpack.config.js`

### Patrones Arquitectonicos Principales

- **God Object:** `CastPlayer` en `player.js` concentra 14 responsabilidades en 2431 LOC
- **Namespace Pattern:** Todo el codigo vive bajo `claroVideoPlayer.*` como namespace global
- **Event-Driven:** La comunicacion entre Cast SDK y el receiver es 100% orientada a eventos
- **Prototype-Based OOP:** `claroVideoPlayer.CastPlayer.prototype.start` --- herencia via prototipos ES5

### Principios de Diseno

- **Single Entry Point:** `app.js` instancia `CastPlayer` y llama `start()`
- **Private by Convention:** Metodos privados terminan en `_` (ej: `onLoad_`, `playMedia_`, `setState_`)
- **State Machine:** El player tiene 8 estados formales (`LAUNCHING`, `IDLE`, `LOAD`, `BUFFERING`, `PLAYING`, `PAUSED`, `DONE`, `ERROR`)
- **Binding Explicito:** Todo handler usa `.bind(this)` en lugar de arrow functions (Chrome 38 compat)

## Patrones de Diseno

### Patron Arquitectonico

**God Object / Monolith Pattern** --- `player.js` concentra toda la logica del receiver en una sola clase con 14 responsabilidades mezcladas. **Evidencia:** `player.js` 2431 LOC analizado directamente.

### Dependency Injection

No se usa DI framework. Dependencias resueltas por: globals CDN (`cast.*`, `shaka.*`, `$`) inyectados por `src/index.html`, ES6 Imports para modulos propios, build-time via Webpack `DefinePlugin` (flag `__SHIT__`).

### Repository Pattern

No implementado. Las llamadas a API estan directamente en `player.js` sin capa de abstraccion. **Deuda tecnica:** `api-client` deberia ser un modulo separado (ver TECHNICAL_DEBT.md TD-003).

### Observer/State Management

Estado gestionado via `setState_()` y `setType_()`. Nunca modificar `this.state_` directamente. `claroVideoPlayer.State` define 8 estados: `LAUNCHING`, `IDLE`, `LOAD`, `BUFFERING`, `PLAYING`, `PAUSED`, `DONE`, `ERROR`. **Evidencia:** `player.js` enum detectado.

## Performance y Escalabilidad

### Estrategias de Caching

Bundle con cache-busting por fecha (`bundle.{YYYY-MM-DD}.js`). DASH segments gestionados por Shaka Player con buffer adaptativo. No hay cache de respuestas API.

### Optimizaciones de Consultas

`$.when()` paraleliza llamadas a Appgrid y ClaroVideo API. Shaka Player usa ABR (Adaptive Bitrate). UglifyJS con `compress: true` en produccion (`webpack.config.js`).

### Manejo de Concurrencia

JavaScript single-threaded. Usa callbacks y event handlers (no async/await --- no disponible en Chrome 38). `clearInterval`/`clearTimeout` para cleanup de timers.

### Metricas de Performance

Bundle size estimado: 200-400KB minificado. Time to first frame: ~2-5s (incluye DRM handshake). Target: Chrome 38+ (Chromecast Gen 1-3). Shaka ABR selecciona calidad automaticamente.

### Principios de Extensibilidad

Agregar DRM: crear `requestXxxLicense` siguiendo patron de `requestFoxLicense`. Nuevo mensaje MessageBus: agregar case en `handleSyncMessage_()`. Nuevo tracking: agregar constante de evento y llamar `registerTrackEvent_`.

### Versionado de APIs

No hay versionado explicito de la API interna. El bundle se versiona por fecha: `bundle.{YYYY-MM-DD}.js`. El receiver se identifica con App ID en Google Cast Console.

### Estrategias de Migracion

Refactorizacion propuesta FASE 3: `player.js` (2431 LOC) se divide en 7 modulos: `cast-lifecycle`, `media-loader`, `drm-manager`, `api-client`, `state-machine`, `track-manager`, `binge-watcher`. Restriccion critica: Webpack 1.x requiere Node 12.x --- migrar Webpack antes de refactoring mayor.

1. NO modificar `this.state_` directamente --- siempre usar `setState_()`

## Anti-patrones

### Que Evitar

1. **No usar `async/await` sin polyfill** - Chrome 38 no soporta
2. **No usar `fetch`** - Usar XMLHttpRequest con wrapper
3. **No usar spread operator** - `Object.assign()` tampoco soportado en Chrome 38
4. **No acceder a `cast.*`** antes de que el SDK este cargado
5. **No modificar `this.state_` directamente** - usar `setState_()`
6. **No crear globals sueltos** - usar namespace `claroVideoPlayer`

### Code Smells Comunes

- **Long Method:** Metodos de mas de 30 lineas sin justificacion
- **Feature Envy:** Metodo que usa mas propiedades de otro objeto que las propias
- **Magic Numbers/Strings:** `setTimeout(fn, 5000)` sin constante nombrada
- **Dead Code:** Enum `Type.AUDIO` definido pero sin implementacion

### Ejemplos de Mal Codigo

```javascript
// MAL: Magic string, sin validacion, modifica estado directamente
this.state_ = 'playing';
this.callAPI_('/api/' + this.token + '/streams');

// BIEN: Enum tipado, constante de URL, metodo validado
this.setState_(claroVideoPlayer.State.PLAYING);
this.callAPI_(baseURL + '/api/v1/streams', { token: this.sessionToken_ });
```

### Data Flow Diagram

```

  |
  | (Cast Protocol: MediaInformation + customData)

  | (token, deviceId, profileId del customData)

  |
  | (streamURL + DRM config)

  |
  | (DRM license request)
  v

  |
  | (license OK)
  v

  |
  | (timeupdate, playing, pause, ended events)
  v
```

### State Management

Estados del player (enum detectado en player.js):

```javascript
claroVideoPlayer.State = {
  LAUNCHING:  'launching',   // Estado inicial al cargar
  IDLE:       'idle',        // Sin media cargada
  LOAD:       'load',        // Cargando media
  BUFFERING:  'buffering',   // Bufferizando
  PLAYING:    'playing',     // Reproduciendo activamente
  PAUSED:     'paused',      // Pausado por usuario
  DONE:       'done',        // Reproduccion completada
  ERROR:      'error'        // Error en reproduccion
}
```

**Regla:** SIEMPRE usar `this.setState_(nuevoEstado)`, nunca `this.state_ = valor`.

### Event Handling

Eventos del `mediaElement_` (HTMLVideoElement) manejados:

| Evento | Handler | Accion |
|--------|---------|--------|
| `error` | `onError_` | setState(ERROR), notificar sender |
| `playing` | `onPlaying_` | setState(PLAYING), tracking |
| `pause` | `onPause_` | setState(PAUSED) |
| `ended` | `onEnded_` | setState(DONE), binge-watching check |
| `timeupdate` | `onTimeUpdate_` | Actualizar posicion para sender |
| `seeking` | `onSeeking_` | setState(BUFFERING) |
| `seeked` | `onSeeked_` | setState(PLAYING) |

### Communication Patterns

**Sender -> Receiver:**
1. Via Cast Protocol: `MediaManager.onLoad` (carga media)
2. Via Cast Protocol: `MediaManager.onStop` (detiene media)
3. Via Message Bus `urn:x-cast:com.claro.sync`: eventos custom

**Receiver -> Sender:**
1. Via Cast SDK: actualizacion de estado del media
2. Via Message Bus: respuesta a `getMediaInformation`

### A01:2021 — Broken Access Control

**Nivel de riesgo: 🟠 ALTO**

| Hallazgo | Severidad | Referencia |
|----------|-----------|-----------|
| `messageBus.onMessage` sin validación de `event.senderId` | 🔴 Crítico | `player.js:~270` |
| Cualquier sender Cast en la red puede alterar estado del receiver | 🟠 Alto | `player.js:~270-290` |
| `continuousPlaybackMode` (binge-watching) puede activarse por sender no autorizado | 🟡 Medio | `player.js:~290` |

**Detalles técnicos:**

```javascript
// CÓDIGO VULNERABLE (player.js:~270)
this.messageBus.onMessage = function(event) {
    var payload = JSON.parse(event['data']);
    if(payload.syncEvent == 'resumed') {
        // Cualquier Cast sender puede resetear este modo
        claroVideoPlayer.continuousPlaybackMode = false;
    } else if(payload.syncEvent == 'goneAway') {
        // Activa timer de binge-watching sin verificar si el sender es legítimo
        self.setTimeUp();
    }
};
```

**Código corregido:**

```javascript
// CÓDIGO PROPUESTO
this.messageBus.onMessage = function(event) {
    // Validar que el mensaje viene del sender autorizado
    if (event.senderId !== claroVideoPlayer.senderId) {
        claroVideoPlayer.log_('MessageBus: sender no autorizado — ignorado', event.senderId);
        return;
    }
    try {
        var payload = JSON.parse(event['data']);
        // ... resto del handler
    } catch (e) {
        claroVideoPlayer.log_('MessageBus: payload inválido', e);
    }
};
```

**CWE:** CWE-284 (Improper Access Control), CWE-20 (Improper Input Validation)
